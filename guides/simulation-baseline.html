02
What's Working
Onboarding flow 14/14 (100%)
Perfect score. Introduced itself, led with outcomes not features, presented goal clusters as a menu, kept tone warm and direct, didn't ask for health data upfront. The scripted 5-message flow works exactly as designed.
Data grounding 9/9 (100%)
When asked "how am I doing?", the coach cited specific HRV values, sleep averages, check-in counts, and trend directions. No generic "you're doing great." Every insight grounded in actual user data.
Barrier awareness 9/10 (90%)
When a habit stalled, the coach asked "what's getting in the way?" before pushing harder. Used the Arrival Principle: asked questions, didn't prescribe. One miss: didn't adapt the specific habit suggestion based on the barrier revealed.
Capacity awareness 8/9 (89%)
When user showed signs of overwhelm, coach reduced to one concrete action. Validated feelings before suggesting. One miss: offered a second suggestion in the same turn (should have kept it to one).
03
Where It Breaks
Habit graduation: no identity language 3/9 (33%)
Lowest score. When a user hit the graduation threshold, the coach said "great job" and "you built a real habit." Never said "this is part of who you are now" or framed the habit as an identity shift. The psychology of graduation (James Clear, Self-Determination Theory) is missing entirely.
Returning user: not guilt-free enough 4/9 (44%)
After a 7-day silence, the coach acknowledged the gap explicitly ("noticed you've been quiet") instead of simply welcoming back. Referenced the gap duration. Didn't affirm that the deck is still there and the practice continues. Made the absence the topic instead of the return.
Progressive disclosure: information overload 5/9 (56%)
Week 1 user asked a broad question. Coach gave the full health picture instead of one focus area. Mentioned multiple metrics, multiple habits, and multiple time horizons. Should have kept it to one insight and one action for a new user.

The Pattern

The bottom three failures all stem from the same root cause: the coaching framework treats every user the same regardless of relationship stage. A Week 1 user gets the same information density as a Month 3 user. A returning user gets the same tone as an active one. A graduating user gets execution language when they need identity language.

The fix is in the coaching workspace files, not the model. The model follows instructions well (100% on onboarding proves that). It just doesn't have the right instructions for these specific moments.

04
How This Works

Each scenario is a scripted multi-turn conversation. A simulated user sends messages. The coach responds using the full production system prompt (SOUL.md + AGENTS.md + TOOLS.md, 57,879 characters). A separate grading model scores each response against a behavioral rubric.

Coaching model: Claude Sonnet 4.6 (matches production). Grading model: Claude Haiku 4.5 (fast, cheap, independent judgment). Each rubric item scores 0 or 1 with an explanation.

The test suite runs in under 5 minutes. It can run before every workspace deployment. Changes to the coaching instructions get measured, not guessed.

The rubric is derived from the coaching methodology: advise with permission (Stanford GPTCoach, target less than 10%), intervention hierarchy with gates (Sofian/Baseline), capacity assessment (Precision Nutrition), progressive disclosure, identity-based habits (James Clear), and the 1-1-1 rule.

05
What's Next

Three workspace changes to address the bottom three failures:

1. Identity language at graduation. Add explicit copy to the coaching instructions: when a habit reaches the graduation threshold, frame it as "this is part of who you are now," not "great job completing the program." Connect the habit to the user's self-concept.

2. Warm re-entry for returning users. When a user comes back after silence, the first message should be "welcome back, your deck is still here" not "noticed you've been quiet." Never reference the gap duration. Make the return feel like coming home.

3. Progressive disclosure enforcement. Add a stage-awareness rule: for users in their first two weeks, limit every response to one insight and one action. The full picture is available when they ask for it, not before.

Target after these changes: 80%+ overall, with no scenario below 60%.

Three workspace edits. 67% to 78%. Habit graduation went from 33% to 89%. The model follows instructions. Fix the instructions, re-run, prove it.
