Google Drive Backup Guide

Back up your entire Google Drive to an external SSD, then clean up storage.

What You Need

  • A Mac (Intel or Apple Silicon)
  • An external SSD (any size that fits your Drive)
  • Python 3 installed. Check by opening Terminal and typing python3 --version
  • A Google account with Drive files to back up

Step 1: Format Your External Drive

  1. Plug in the drive
  2. Open Terminal (Cmd + Space, type "Terminal")
  3. Check the drive is visible: 
    ls /Volumes/
    You should see your drive name in the list.
  4. Find your drive's disk number: 
    diskutil list external
    Look for your drive (it will show the size). Note the /dev/diskN number.
  5. Format it for Mac: 
    diskutil eraseDisk APFS "My Backup" /dev/diskN
    Replace /dev/diskN with your actual disk number. This erases everything on the drive.

Step 2: Enable Google Drive API

  1. Go to console.cloud.google.com
  2. Create a new project (or use an existing one)
  3. Go to APIs & Services > Library
  4. Search for "Google Drive API" and click Enable
  5. Go to APIs & Services > Credentials
  6. Click Create Credentials > OAuth client ID
  7. Application type: Desktop app
  8. Name it whatever you want
  9. Download the JSON file, rename it to credentials.json
If this is your first time setting up a GCP project, you will also need to configure an OAuth consent screen. Choose "External" and fill in the basics. You only need to add yourself as a test user.

Step 3: Set Up the Backup Tool

Open Terminal and run these commands one at a time:

mkdir ~/drive-backup
cd ~/drive-backup
python3 -m venv .venv
.venv/bin/pip install google-api-python-client google-auth-httplib2 google-auth-oauthlib

Copy credentials.json into the ~/drive-backup/ folder.

Download the backup script and save it as drive_backup.py in the same folder. (Andrew will send you this file.)

Step 4: Authenticate

The first time you run the backup, a browser window will open asking you to sign in to Google and grant access. Sign in with the account you want to back up.

This only happens once. After that, the tool remembers your login.

Step 5: Run the Backup

cd ~/drive-backup
.venv/bin/python3 drive_backup.py "/Volumes/My Backup/Google Drive Backup"

Replace "My Backup" with whatever you named your drive in Step 1.

The script will:

  • Map all your folders
  • Download every file, keeping the folder structure
  • Export Google Docs as PDF, Google Sheets as Excel
  • Skip files already downloaded (safe to re-run if interrupted)
  • Log everything to _backup_log.jsonl
This can take a while depending on how much you have. It is safe to stop (Ctrl+C) and restart. It picks up where it left off.

Step 6: Verify

When it finishes, you will see a summary:

BACKUP COMPLETE
  Downloaded: 13967
  Skipped: 500
  Errors: 13
  Total size: 19.60 GB

A few errors are normal. These are usually files shared with you that the owner restricted, or Google Forms that cannot be exported.

Check the total size on the drive:

du -sh "/Volumes/My Backup/Google Drive Backup/"

What to Keep vs. Delete from Drive

Keep in Drive

  • Documents you collaborate on with other people
  • Anything shared with a team or group
  • Files you access from your phone
  • Your resume or docs you send links to

Safe to Delete (now backed up)

  • Old course materials, ebooks, PDFs
  • Archives and one-time downloads
  • Photos and videos also in Google Photos
  • Browser cache archives, old app backups
  • Anything you have not opened in 6+ months

Check Before Deleting

  • Shared folders where you are the owner. If you delete them, other people lose access.
  • Google Forms. Responses are tied to the form and do not export the same way.

Clean Up (Optional)

Only do this after you have verified the backup on your external drive.

Easiest way: go to drive.google.com, click Storage in the left sidebar (sorts by file size), and delete the biggest files first. Empty trash after.
