Integration Trailheads

Six places the coaching layer connects to the existing app. Organized by build order — Today tab first (no prerequisites), then Pulse.

Converge summary: The coaching engine has depth (21-day sandbox, phases, weekly reflections, evidence protocols). But it doesn't need to ship all at once. These six trailheads are the surfaces where coaching intelligence becomes visible to the user. Each one is a small, shippable piece.

How I got here: Ran through Paul's latest main build. Mapped every screen where the coaching layer could surface without breaking what's already working. Cross-referenced with IRL debrief (small iterations, converged work, "wow" gap). Each trailhead is 1-2 PRs.

Design intent

Adds reflection depth without adding friction. The emoji response takes 1 second. The optional free-text field ("Anything else?") captures richer context — voice input makes this nearly zero-effort. Always skippable — the one-tap path still works.

Sentiment as signal layer (new — not yet in coaching engine)

Emoji + free-text over time becomes a sentiment signal that powers multiple coaching decisions:

  • Anchor habit detection: Consistently "great" ratings identify the habit that fits your life best. Protect it. This is the one to build layers around.
  • Struggle detection: Persistent "tough" ratings trigger coach intervention — rescope the habit, suggest group accountability, or bench it before it drags other habits down.
  • Lifestyle fit scoring: Sentiment + consistency together reveal whether a habit is sustainable identity-level or just grinding. A habit with perfect streaks but "meh" ratings is fragile.
  • Seasonal adaptation: As your health baseline rises and more habits graduate to practicing, sentiment reveals which layers still bring energy vs. which are coasting. Priorities shift by season — the data shows when.

Requires new field on CDCheckIn (reaction enum + optional note). Currently the model has: id, date, completed, note. The coaching engine (AIService) uses only behavioral signals today — this would be the first qualitative input fed to the AI prompt context.
