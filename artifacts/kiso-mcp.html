Design Decision

Let Any Claude Talk to Your Health Data

A thin MCP wrapper that connects Claude Desktop, Claude Code, or any MCP client to the Kiso health intelligence API. Your data, your Claude, one config line.

The Idea

Alma ships an MCP integration that lets Claude read your nutrition data. We do the same thing, but for your entire health picture: labs, wearables, habits, meals, weight, coaching context. You add one server to your Claude config and suddenly Claude knows your numbers.

What This Feels Like

"What's my HRV trending this week?" "Log my lunch: chicken bowl, 650 cal, 55g protein." "How did my sleep affect my recovery score?" All from Claude, no app switching.

The Architecture

How It Connects

Your Claude (Desktop or Code)
    |
    | MCP (stdio)
    v
kiso-mcp-server (~200 lines, thin wrapper)
    |
    | HTTPS + token auth
    v
Kiso API (auth.mybaseline.health)
    |
    v
Your data (SQLite + CSVs, per-user isolation)

The MCP server is a thin wrapper. It translates tool calls into REST API calls with your auth token. No business logic in the wrapper. All intelligence stays in Kiso.

Setup

Add to your .mcp.json

{
  "kasane": {
    "command": "python3",
    "args": ["-m", "kasane_mcp"],
    "env": {
      "KASANE_API_URL": "https://auth.mybaseline.health",
      "KASANE_TOKEN": "your-token-here"
    }
  }
}

One config block. Then Claude knows your health data.

The Bigger Picture

MCP is how health data stops being trapped in apps. Once your labs, wearables, and habits flow through a standard protocol, any AI can reason about your health. The coaching layer is what turns that reasoning into behavior change. Data without action is a dashboard. Data with coaching is a program.
