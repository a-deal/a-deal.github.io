LearnAIR Curriculum Analysis + Our Augmentation

What the Foundation Series teaches, what’s missing, and how we extend it.

March 26, 2026

What They Teach

The Foundation Series is 3 live sessions × 1.5 hours each. Groups of 5–20. Here’s what each session covers.

Session 1
Orientation + Personalization
  • What ChatGPT is (library metaphor)
  • Interface tour
  • Memory architecture (3 tiers)
  • DIRECT prompting framework
  • Digital twin + counterbalance persona
  • Data privacy settings
  • Scheduled tasks
Session 2
Organization + Personas
  • Progress check-ins
  • Thread management challenges
  • Projects feature (files, sharing)
  • App integrations (Drive, Gmail, Slack)
  • Persona creation framework
  • Live exercise: build a persona (18 min)
Session 3
Custom GPTs + Agent Mode
  • AI-powered app development
  • Agent mode demo
  • Custom GPT architecture
  • Building “digital employees”
  • 8,000 char limit, knowledge base
  • Macro vs Micro GPT strategy
  • Troubleshooting

What They Do Well

  1. Accessible metaphors. “World’s library with an intelligent librarian” lands for non-technical people. The “digital employee” framing gives people a mental model they can work with.
  2. Structured prompting. The DIRECT framework is solid. Giving people a recipe for prompts removes the blank-page anxiety.
  3. Persona + counterbalance. Creating a digital twin and then a counterbalance that fills gaps is a genuinely useful pedagogical move.
  4. Hands-on time in session. 18 minutes of live building in Session 2. People leave having done something, not just heard about it.
  5. Macro vs Micro GPT strategy. The org chart metaphor (Sales GPT, Marketing GPT at macro; email writer, LinkedIn post creator at micro) is a good way to think about organizing AI tools.
  6. Real participant wins. John completed 4–6 months of work in one week. Cindy built an HR compliance hub. Shannon streamlined a 6–9 month framework creation to days. These are real outcomes.

The Gap

Eight things the Foundation Series doesn’t cover. These aren’t criticisms — they’re the natural next layer that a 4.5-hour introductory course can’t reach.

1
No organizational coordination layer
Everything is individual. “Build YOUR persona.” Zero discussion of what happens when 50 people each do this. Who coordinates? Who validates? Who governs?
2
ChatGPT-only, platform-locked
Entire curriculum is OpenAI-specific. No model-agnostic thinking. No Claude, Gemini, or open-source. Students are locked into one vendor.
3
No file system / version control
They teach ChatGPT Projects (folders) but not actual file management. No git, no version history. JB’s Session 2 frustration (copy-paste admin) is a direct symptom.
4
No CLI / terminal exposure
Everything is GUI. No command line, no voice-first workflow, no understanding of why the terminal matters for managing systems vs. building individual things.
5
No security / governance depth
They mention privacy toggles but not: who reviews AI output? What happens when a Custom GPT gives wrong advice? What’s the approval workflow?
6
No workforce transition framework
A skills course, not a transition program. Nothing about how this changes jobs, teams, or managing the anxiety when AI does 80% of the work.
7
No measurement / outcomes
They cite 92% adoption but teach no framework for measuring ROI. “4–6 months of work in one week” is anecdotal, not systematic.
8
Troubleshooting is reactive, not systemic
Context limits, memory loss, hallucinations are treated as bugs to work around, not architectural realities to design for.

Practical Augmentation Plan

An 8-week applied program. Mike and Andrew bridge the gaps through building, not lectures.

Week 1–2 CLI Foundations
  • Install and configure Claude Code
  • CLAUDE.md as persistent persona (replaces per-session persona loading)
  • Hub framework: research/, decisions/, plans/, meetings/, execution/
  • First real task: take LearnAIR session notes and publish them as a hub page
  • Voice-first workflow: Monologue + terminal
Week 3–4 System Thinking
  • Managing multiple agents (sub-agents, parallel tasks)
  • Version control with Git (own your work, never lose progress)
  • Model-agnostic principles: do the same task in ChatGPT, Claude, and Gemini. Compare.
  • Build a Custom GPT AND a CLAUDE.md for the same purpose. See the difference.
Week 5–6 Organizational Layer
  • What happens when 5 people each build their own tool? Simulate it.
  • Governance: who reviews AI output before it ships?
  • The coordination problem: individual productivity vs. organizational ROI
  • Interview one employer: “If a veteran showed up who could manage your AI operations, would that be valuable?”
Week 7–8 Integration + Pilot Prep
  • Build a LinkedIn content system using the hub as source material
  • Run a mock “AI ecosystem supervision” exercise
  • Document everything: what worked, what broke, what surprised you
  • Produce a case study of the learning journey (Mike’s “oh shit” moments as testimonials)
