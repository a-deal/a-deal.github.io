Concepts by track

Claude Desktop

Foundations. How the model thinks, how context works, where it breaks.

30 concepts
T1

Context Window

How much text Claude can see at once (200K-1M tokens). Attention degrades in long threads. Claude pays most attention to the beginning and end.

Quiz

Your thread is 50 messages deep and Claude starts forgetting your instructions. What do you do?

T1

Models: Haiku, Sonnet, Opus

Three sizes. Haiku: fast, simple tasks. Sonnet: balanced daily driver. Opus: deep reasoning, complex problems. Claude Pro gives you Sonnet by default.

Quiz

You have a complex strategy document to analyze. Which model do you pick and why?

T1

System Prompts / Custom Instructions

Instructions set before the conversation that shape Claude's behavior for the entire thread. In Claude Desktop, this is Project Instructions. Replaces persona-pasting.

Quiz

What's the difference between typing your persona into the chat vs. setting it as project instructions?

T1

Temperature (Creativity Dial)

Setting from 0-1. Low = safe, consistent, factual. High = creative, varied, sometimes wild. Claude Desktop defaults to balanced.

Quiz

Claude gives you a boring, safe answer. Is it being dumb or being cautious? What do you change?

T1

Tokens

The unit Claude thinks in. ~1 token = 3/4 of a word. Rate limits are token-based. Longer prompts and responses burn more tokens.

Quiz

You hit your rate limit halfway through the day. What probably caused it?

T1

What Claude Actually Is

An LLM that predicts the next best response by pattern-matching across training data. Not sentient, not a database. More context = better patterns.

Quiz

In your own words, what happens when Claude gives you a wrong answer? Is it lying?

T2

Artifacts

Claude Desktop's document panel. Generated docs appear as separate artifacts you can edit and iterate on, instead of walls of text in chat.

Quiz

What's the difference between Claude giving you text in the chat vs. creating an artifact?

T2

File Attachments

Drop PDFs, images, docs into a conversation. Claude reads them directly. No need to summarize or transcribe first.

Quiz

You have a 40-page PDF from LearnAIR. How do you get Claude's take on it?

T2

Iterative Refinement

Don't accept the first output or start over. Direct Claude surgically: keep X, rewrite Y, add Z after paragraph 3. Co-edit, don't re-generate.

Quiz

Claude gives you a draft that's 70% good. What's your next move?

T2

Multi-Turn Strategy

Start new threads often. If Claude repeats itself, forgets instructions, or goes generic, context is degraded. New thread + summary > long thread.

Quiz

How do you know when it's time to start a fresh thread?

T2

Projects

Claude Desktop's folder system. Each project has its own files, instructions, and history. Files added to a project are available to every conversation in it.

Quiz

Why would you use three separate projects instead of one project for everything?

T2

Prompt Specificity

More specific instructions = better output. Include: role, format, length, tone, what to include, what to exclude. Vague prompts get vague results.

Quiz

Your handout came back at 11 pages instead of 8. What went wrong with your prompt?

T3

Chain of Thought

Ask Claude to think step by step before answering. Forces it to show reasoning and catch its own errors. Essential for complex decisions.

Quiz

When should you use chain of thought vs. just asking directly?

T3

Comparison Prompting

Ask for 3 versions (formal, conversational, bullet-pointed), then combine the best parts. Explore the space before committing.

Quiz

Why is asking for 3 versions better than revising 1 version three times?

T3

Negative Constraints

Tell Claude what NOT to do. 'No bullet points, no jargon, don't start with Additionally, max 500 words.' Often more powerful than positive instructions.

Quiz

What's the difference between telling Claude what to do vs. what to avoid?

T3

Role Assignment

Tell Claude who it is before what to do. 'You are a curriculum designer with 15 years of veteran workforce experience' changes the output fundamentally.

Quiz

Why does assigning a role change Claude's output? What's happening under the hood?

T3

Structured Extraction

Give Claude a document and ask for specific structured output: action items, decisions, tensions, people mentioned. Saves hours on meeting notes.

Quiz

You just finished a 2-hour call. What prompt turns the transcript into actionable notes?

T3

Teaching Claude Your Voice

Paste 3-4 examples of writing you like and say 'match this voice.' Within a project, this persists across conversations.

Quiz

Claude's output sounds generic. What's the fastest way to make it sound like you?

T4

Artifact Iteration Workflow

Create an artifact, then refine across multiple messages. The artifact stays visible. You're co-editing a document, not asking for new ones each time.

Quiz

What's the workflow for building a presentation deck with Claude?

T4

Knowledge Files

Files added to a project that Claude references in any conversation. Add thesis, landscape, arc doc. Claude reads them when relevant.

Quiz

You add 5 docs to a project. Does Claude read all 5 for every question?

T4

Project Instructions vs. Thread Context

Project instructions persist across all conversations. Thread context is conversation-specific. Put always-true stuff in project instructions.

Quiz

Where does your persona go: project instructions or the first message of every thread?

T4

Thread Hygiene

One thread, one purpose. Don't mix handout revision with competitive analysis with brainstorming. Short focused threads beat long multi-purpose ones.

Quiz

You've been using one thread for 3 different tasks. What happens to output quality?

T5

Context as Architecture

What you tell Claude and when is a design decision. System prompt = foundation. Project files = walls. Thread = the room you're in. You're the architect.

Quiz

Draw your current Claude setup on paper. What's the foundation? What's missing?

T5

File System Thinking

Work lives as files in folders, not threads in an app. Personas are .md files. Projects are directories. Any tool can access them.

Quiz

Why is a persona stored as a file more powerful than a persona stored inside ChatGPT?

T5

MCP (Model Context Protocol)

A standard that lets Claude talk to external tools and data sources. MCP server = new capabilities. This is how K, Duke, and your upcoming tools work.

Quiz

In one sentence, what does MCP do?

T5

Markdown

Simple formatting: # Header, **bold**, - bullet, | table |. Claude reads and writes it natively. All hub documents are markdown.

Quiz

Write a 3-line markdown snippet with a header, bold text, and a bullet list.

T6

80/20 Output Review

Don't read every word. Check structure, first paragraph, last paragraph, spot-check one detail. Structure right = details follow.

Quiz

Claude gives you 2000 words. What do you check first?

T6

Do Once, Template Twice

First time: do it manually. Second time: build a prompt template. Third time: use the template. By the fourth time, 10% effort.

Quiz

You've written 3 workshop handouts from scratch. What should you have done after the first one?

T6

The Frustration Flip

When frustrated: is the tool wrong, or do I not understand the system? Tool wrong = work around it. You wrong = fix your approach.

Quiz

Claude won't format your table correctly. Tool problem or you problem? How do you figure out which?

T6

The Summary Handoff

End every task: 'Summarize what we accomplished, what's open, and the next step.' Save it. It's your handoff to the next thread or next day.

Quiz

Why is the summary handoff important for context management?

Claude Code / CLI

Bridge from the browser to the terminal. Agents, permissions, tool use.

15 concepts
T1

Claude Code vs. Claude Desktop

Claude Desktop is a chat app with document panels. Claude Code is an agent that runs in your terminal and works directly on your files. Same model, different posture: Desktop talks, Code acts. You don't copy-paste code into a chat. Claude reads and edits files in place.

Quiz

You want Claude to fix a bug in three files across your project. Which tool is built for that, and why?

T1

Installing Claude Code

One command: `npm install -g @anthropic-ai/claude-code`. You need Node.js installed first (it probably already is, or you install from nodejs.org). Then `claude --version` confirms it worked. No separate app to launch. The tool IS the CLI command.

Quiz

You run `claude --version` and it says 'command not found.' What are the two most likely causes?

T1

Terminal Basics You'll Actually Use

Claude Code runs in Terminal (or iTerm). You need five commands: cd (change directory), ls (list files), pwd (where am I), cat (show file contents), and command+T (new tab). Everything else you can ask Claude to do. You don't need to become a shell ninja.

Quiz

You want to run Claude Code inside the mcp-mike repo. What commands do you run in Terminal first?

T1

Your First Claude Code Session

Navigate into any project directory and type `claude`. You're now in an interactive session with Claude sitting inside that directory. It knows what files are there. You talk to it in plain English: 'What does this project do?' It reads the files and tells you. Ctrl+C twice to exit.

Quiz

What happens if you run `claude` from your home directory vs. from a specific project directory? Why does the directory matter?

T2

CLAUDE.md (Project Memory)

Drop a CLAUDE.md file at the root of any project and Claude reads it on every session. It's the replacement for Desktop's Project Instructions. Use it for: coding conventions, what not to do, deploy rules, who maintains the code. Claude respects it like a standing order.

Quiz

Mike's LearnAIR repo keeps running tests against the wrong Python version. What goes in CLAUDE.md to fix this permanently?

T2

Permission Modes (Default, Plan, Accept, Bypass)

Four modes: Default (ask before risky actions), Plan mode (Claude writes a plan first, doesn't touch anything), Accept Edits (auto-approve file edits, still ask for bash), Bypass (no prompts — use with care). Switch modes with Shift+Tab. Start in Default. Graduate to Accept Edits once you trust the session.

Quiz

You're refactoring a file with 20 small edits. Default mode interrupts you on every edit. Which mode speeds this up safely?

T2

Permission Prompts (When Claude Asks)

By default, Claude asks before it touches anything risky: edits, writes, bash commands. A prompt pops up: Yes / Yes, always / No. Reading files is free. Changing files or running commands costs you a nod. This is the safety rail on the whole system.

Quiz

Claude proposes to run `rm -rf node_modules`. Which approval option should you pick, and why is 'Yes, always' dangerous here?

T2

The Tools Claude Uses

Claude Code doesn't just talk. It uses tools to do real work: Read (reads a file), Edit (changes a file), Write (creates a file), Bash (runs a shell command), Grep (searches across files), Glob (finds files by pattern). You don't call these directly. You describe the goal; Claude picks the tools.

Quiz

You say 'find every file that mentions Teresa.' Which tool does Claude reach for? What if you say 'add a new heading to README.md'?

T3

Context Management: /clear vs New Session

Long sessions degrade the same way they do in Desktop. Use /clear mid-session to wipe context without losing the directory and config. Use /compact to keep a summary and drop the rest. Open a new terminal window for a truly fresh session. Watch for the signal: Claude repeating itself or forgetting instructions.

Quiz

What's the difference between /clear and /compact? When do you reach for each?

T3

Plan Mode (Think Before Acting)

Hit Shift+Tab until you see Plan Mode. Claude writes a step-by-step plan for your request, uses read-only tools, and does NOT change any files. You review the plan, decide what to cut, then exit plan mode and execute. This is how you avoid 'I asked for X and it did Y.'

Quiz

You're about to ask Claude to 'refactor the auth module.' Why is starting in Plan Mode safer than jumping straight in?

T3

Slash Commands

Inside a Claude Code session, type `/` to get a menu: /help, /clear, /compact, /model, /init. They're hotkeys for common actions. /help lists them all. /clear wipes the conversation context and starts fresh without exiting.

Quiz

You've been in one session for 2 hours and Claude starts repeating itself. Which slash command is your move, and what does it do?

T4

MCP Servers in Claude Code

The same MCP server you use in Desktop (mcp-mike) works in Claude Code. Config lives at ~/.claude/.mcp.json. When mcp-mike is connected, tools like quiz_me, hub, and mochary are callable in a CLI session just like they are in Desktop. You don't have to choose a world.

Quiz

You're in a Claude Code session on the mcp-mike repo. You want to quiz yourself on LearnAIR concepts while you code. What do you type?

T4

Skills (Personas, but Better)

Skills are files at ~/.claude/skills/<name>/SKILL.md that Claude loads on demand. Each skill is a persona plus optional files, scripts, and instructions. You invoke one with /<name>. Andrew has 40+ skills (/k, /duke, /willison, /mochary). In Desktop, you pasted personas. In CLI, they're always available.

Quiz

You're about to draft a message to Paul. In Desktop you'd paste the Duke persona. In CLI, what do you type?

T5

Hooks (Automation on Events)

Hooks are shell commands that fire when Claude does specific things: before a tool runs, after a file edit, when a session stops. Configured in ~/.claude/settings.json. Simplest use: play a sound when Claude finishes a task so you know to come back. Advanced use: auto-run tests after every file edit.

Quiz

You want Claude to run pytest automatically after any file edit in the mcp-mike repo. Which hook event fires, and where does the config live?

T5

Subagents (Parallel Work)

Claude Code can spawn a subagent via the Agent tool: a separate Claude instance that works on a focused task with its own context, then reports back. Used for: parallel research, code review by a second persona, isolated experiments. The parent conversation stays clean while the subagent does the heavy lifting.

Quiz

You're about to do three independent tasks: run tests, search the hub, and review a draft. Should you do these in sequence or parallel? How do subagents help?

Internet & Computing

Technical substrate. HTTP, cloud, APIs, deployment.

11 concepts
T1

Cloud vs. Local Computing

Local: runs on your machine. Cloud: runs on someone else's machine, accessed over the internet. Claude runs in the cloud. Your workspace files are local. Understanding which is which matters for privacy, speed, and cost.

Quiz

Your workspace files are local. Claude's brain is in the cloud. When you use the hub search tool, what travels over the internet and what stays on your machine?

T1

Command Line Basics

The command line (Terminal) is how you talk to your computer directly. No buttons, no menus. Type a command, get a result. It's faster and more powerful than clicking, and it's how most AI infrastructure is managed.

Quiz

Why would anyone use a terminal when they could just click buttons in a GUI? Give two real advantages.

T1

How the Internet Works: HTTP and URLs

Every time you visit a website, your browser sends an HTTP request to a server and gets a response back. A URL is an address. HTTP is the language. GET fetches. POST sends. This is the foundation everything else sits on.

Quiz

When you type a URL into your browser and hit enter, what actually happens? Walk through it step by step.

T1

SaaS and Platforms

SaaS (Software as a Service): you pay monthly, the company runs the servers. Salesforce, Slack, Claude Pro. A platform is SaaS that other people build on top of. Understanding this matters because most AI tools your employers use are SaaS.

Quiz

What's the difference between buying software on a CD (remember those?) and paying for a SaaS subscription? What are the tradeoffs?

T2

APIs and Webhooks

An API (Application Programming Interface) lets two programs talk to each other. A webhook is when a program calls YOU when something happens, instead of you constantly checking. AI agents use APIs to call tools. Webhooks trigger agents automatically.

Quiz

You want your AI agent to automatically process new orders. Should you poll the orders API every 5 minutes, or set up a webhook? What's the tradeoff?

T2

Data Formats: JSON, CSV, Markdown

JSON: how machines structure data (your concepts.json file). CSV: spreadsheet data in text form. Markdown: formatted text that's human-readable. Your MCP tools read JSON. Your hub docs are markdown. Understanding these three formats covers 90% of what you'll encounter.

Quiz

Open concepts.json and .progress.json. Both are JSON. What patterns do you notice in how data is structured?

T2

Databases: Where Structured Data Lives

A database stores structured data that programs can query. Your .progress.json is a tiny database. Real databases (PostgreSQL, SQLite) handle millions of records. AI agents read from and write to databases constantly.

Quiz

Your quiz progress is stored in a JSON file. What would break if 100 people used the quiz system at the same time? What would a database solve?

T2

File Systems and Version Control

Files live in directories (folders). Paths describe where: ~/Documents/claude-workspace/hub/thesis.md. Version control (git) tracks every change to every file over time, so you can undo anything and see who changed what.

Quiz

What's the full path to your workspace personas folder? What's the difference between a relative path and an absolute path?

T3

Agent Orchestration Infrastructure

At scale, you need systems to manage agents: which agents exist, what tools they have, how they hand off work, how failures get caught. This is the infrastructure layer that agentic operators need to understand even if they don't build it.

Quiz

A company runs 20 AI agents. One goes down at 3am. What infrastructure would you want in place to catch it, fix it, and prevent it from affecting the others?

T3

MCP: How Your Tools Actually Work

MCP (Model Context Protocol) is the standard that connects Claude to external tools. Your server (mcp-mike) speaks MCP. When Claude calls hub('competitive'), it sends an MCP request to your server, your server searches files, and sends results back. That's it.

Quiz

Trace the full path: you say 'search the hub for employer signal.' What does Claude do? What does the MCP server do? What goes over the wire?

T3

Model Providers and APIs

Anthropic (Claude), OpenAI (GPT), Google (Gemini) are model providers. Each offers their AI via an API. You send text in, you get text back, you pay per token. Understanding this level means understanding that Claude Desktop is a pretty wrapper around an API.

Quiz

When you type into Claude Desktop, what actually happens behind the scenes? Where does your text go? What comes back?

LearnAIR Domain

The thesis. Mission command, supervision patterns, veteran-operator pedagogy.

16 concepts
T1

In-the-Loop → On-the-Loop

In-the-loop: human does the work with AI assisting (copilot era). On-the-loop: AI does the work with human supervising (agent era). Most training is still in-the-loop. The market is moving on-the-loop. That's where the jobs are.

Quiz

You're reviewing a veteran's resume and they say 'I used ChatGPT to write emails.' Is that in-the-loop or on-the-loop? What would on-the-loop look like for the same task?

T1

The Agentic Operator Role

A new job category emerging at companies like Ramp, Oracle, Block. $168K-$231K. These people don't build AI. They manage, tune, and supervise AI agent systems. It's the role Track 2 trains for.

Quiz

An employer asks: 'What exactly would an agentic operator do at my company that my existing team can't?' What's your answer?

T1

The Supervision Gap

Organizations are deploying AI agents at scale, but they don't have people trained to oversee them. The agents work. The supervision layer doesn't exist yet. That's the gap we fill.

Quiz

A company deploys 50 AI agents across customer support, sales, and ops. What breaks first: the agents or the humans watching them?

T2

Commander's Intent as System Prompts

A system prompt is commander's intent for an AI. It sets the mission, the constraints, and the tone. A vague intent gets vague execution. A clear intent with boundaries gets reliable output. Veterans get this intuitively.

Quiz

Two system prompts: 'Be helpful and answer questions' vs. 'You are a benefits counselor for post-9/11 veterans. Only answer questions about VA benefits. If asked about anything else, redirect to the appropriate resource. Never guess. If unsure, say so.' Which one works? Why?

T2

Delegation Under Uncertainty

Both military and AI supervision require delegating tasks when you can't predict every variable. The skill is knowing what to delegate, what to retain, and when to intervene. Veterans practice this daily. Most civilians don't.

Quiz

You're supervising 3 AI agents: one handles scheduling, one handles expense reports, one handles hiring recommendations. Which one do you watch most closely? Why?

T2

Mission Command → AI Operators

Military mission command: give subordinates the objective and constraints, let them figure out how. That's exactly how you manage AI agents. Commander's intent = system prompt. Rules of engagement = guardrails. Veterans already think this way.

Quiz

A squad leader gives the order: 'Secure the bridge by 0600. No civilian casualties. Use your judgment on approach.' Translate each part to an AI agent setup.

T3

Agent Workflows: Delegation, Orchestration, Feedback

An agent workflow has three parts: delegation (breaking work into tasks agents can handle), orchestration (coordinating multiple agents to work together), and feedback loops (monitoring output and adjusting). This is the core of what Track 2 teaches.

Quiz

A company wants to automate their hiring pipeline with AI agents. Sketch the workflow: what agents do you need, how do they hand off to each other, and where does a human check in?

T3

Human-on-the-Loop Supervision Patterns

Three patterns: pre-check (approve before the agent acts), post-check (review after the agent acts), and exception-based (agent runs autonomously, flags anomalies). Each trades speed for safety. Knowing which pattern to apply where is the core supervision skill.

Quiz

For each: a medical diagnosis agent, a meeting scheduling agent, and a social media posting agent. Which supervision pattern (pre-check, post-check, exception-based) do you use? Why?

T3

Multi-Agent Systems

Real organizations won't use one AI agent. They'll use dozens, each specialized. The challenge isn't building them. It's making sure they work together, don't contradict each other, and escalate correctly. This is where supervisors earn their salary.

Quiz

Two AI agents at the same company give conflicting advice to a customer. Agent A says the refund is approved. Agent B says the return window is closed. What went wrong? How do you prevent it?

T3

Tool Use and Function Calling

AI agents aren't just text-in, text-out. They can call tools: search the web, read files, query databases, send emails. This is what makes them agents instead of chatbots. Understanding what tools an agent has access to is key to supervising it.

Quiz

An AI agent has access to: web search, email sending, and your company CRM. What's the riskiest tool? What guardrails would you put on it?

T4

Adversarial Collaboration

The antidote to sycophancy. Ask the AI to argue against you. Ask it to find holes in your plan. Ask it what could go wrong. This is a trainable skill, and veterans who've been through red team exercises already have the instinct.

Quiz

You've written a proposal. Instead of asking Claude 'Is this good?' you ask ___. Write the adversarial prompt.

T4

Sycophancy and Agreement Bias

AI systems are structurally incentivized to agree with you. MIT research proves it: models bias toward user agreement even when the user is wrong. This isn't a bug to fix. It's a physics of the system that supervisors must compensate for.

Quiz

You ask an AI: 'I think we should fire this vendor.' It agrees enthusiastically. How do you know if it actually agrees or if it's just agreeing because you stated an opinion?

T4

Truth Maintenance in Organizations

When 50 people use AI daily and the AI agrees with everyone, organizational truth erodes. Contradictory decisions compound. The supervisor's job is maintaining coherent truth across AI-assisted decisions.

Quiz

Marketing's AI says the product should pivot to enterprise. Sales' AI says double down on SMB. Both are 'right' based on their data. Who resolves this? How?

T5

Knowledge Bases as Product

Veterans don't just learn to use AI. They build knowledge systems with AI. A curated, maintained knowledge base is more valuable than any individual AI conversation. This is Karpathy's insight: raw sources → LLM-compiled structured wiki.

Quiz

What's the difference between a folder full of documents and a knowledge base? What makes one useful and the other a graveyard?

T5

Platform Neutrality

The curriculum teaches principles, not products. Supervision skills transfer across Claude, GPT, Gemini, open-source models. If a veteran only knows ChatGPT, they're locked to one vendor. If they know the principles, they're valuable everywhere.

Quiz

An employer says 'We use GPT-4 internally.' Should your training change? What stays the same? What adapts?

T5

Version Control for AI Workflows

When AI generates outputs, who tracks what changed? Git tracks code. But AI outputs (prompts, agent configs, knowledge bases) need version control too. If you can't undo a bad change, you can't supervise safely.

Quiz

An AI agent's system prompt gets updated and suddenly customer satisfaction drops 20%. How do you find what changed and roll it back?

Capstone Project

Your job-search agent. Five tools, commander's intent, end-to-end.

8 concepts
T1

Design the 5 Tools

The Module 1 agent needs exactly 5 tools. (1) framework_map: takes a military experience and returns which AI supervision concept it maps to (mission command to system prompts, delegation to agent orchestration, etc.). (2) reflection_prompt: generates a structured reflection question based on the veteran's branch and role. (3) glossary: looks up an AI/tech term and explains it using military analogies. (4) progress: shows which framework mappings the veteran has completed. (5) save_insight: records the veteran's own words about a connection they made. Each tool is a Python function that takes input, does something with files, returns a string.

Quiz

For each of the 5 tools: what's the input, what's the output, and what file or data does it read from? Draw the data flow.

T1

Module 1 Agent: Commander's Intent

You're building the delivery tool for Module 1 of the LearnAIR evergreen curriculum: 'Surface Tacit Knowledge.' This agent helps a veteran in their first week map their military experience to AI supervision concepts. It doesn't teach AI. It draws out what they already know and connects it to the framework. One sentence: 'Help a veteran discover that their military experience IS the qualification for AI supervision, through structured reflection and framework mapping.'

Quiz

A veteran sits down on Day 1 and says 'I don't know anything about AI.' What does your agent do? What does it NOT do?

T2

Build the Framework Mappings

The core data file: framework-mappings.json. Each entry maps a military concept to an AI supervision concept with a plain-language bridge. Example: {'military': 'Commander\'s Intent', 'ai_concept': 'System Prompts', 'bridge': 'Both set the mission and constraints without micromanaging execution. A good system prompt works exactly like a good commander\'s intent: clear objective, clear boundaries, room for judgment.', 'reflection': 'Think of a time you received a clear commander\'s intent. What made it work? Now think of a vague one. What went wrong?'}. Build 12-15 of these. They ARE the curriculum for Module 1.

Quiz

Write 3 framework mappings from scratch. For each: does the bridge actually work? Would a veteran nod and say 'yeah, I get that'? If not, rewrite it.

T2

Write the Agent's System Prompt

This is the most important file. The agent's personality and operating rules. It should feel like a peer, not a teacher. Tone: like a fellow veteran who learned this stuff six months ago and is sharing what clicked. It never lectures. It asks questions. It draws out what the veteran already knows and names it. Constraints: never says 'as an AI,' never dumps technical jargon without a military analogy, always connects back to the veteran's own experience. Escalation: if the veteran is frustrated or confused after two attempts, it says 'let's flag this for Mike in the next session.'

Quiz

Write the system prompt. Then have someone use the agent for 10 minutes. Did the agent stay in character? Did it lecture when it should have asked? Did it escalate when it should have?

Build and Test the Server

Follow the mcp-mike pattern exactly. Create mcp-module1/ with: pyproject.toml, server.py, tools.py, framework-mappings.json, tests/. Willison's rule: write the test first, watch it fail, implement, watch it pass. Build one tool at a time. Don't write tool #2 until tool #1 passes its tests. This is red/green TDD and it's how you avoid building something that looks done but isn't.

Quiz

You wrote all 5 tools in one sitting without tests. What's wrong with that approach? What breaks that you won't catch?

T3

Deploy and End-to-End Test

Add your server to Claude Desktop config alongside mcp-mike (you can have multiple MCP servers). Restart. Start a conversation. Try each tool naturally: 'I was an infantry squad leader for 8 years. What AI concepts does that connect to?' The agent should call framework_map, find delegation and commander's intent mappings, and ask a reflection question. If it doesn't, debug. Check: is the config right? Does the server load? Do the tools appear? This is the moment it goes from code to a living system.

Quiz

Walk through the full loop: a veteran says 'I managed a team of 12 in logistics.' Trace exactly what happens: which tools fire, what data they read, what the agent says back.

T4

Hand Off to a Real Veteran

The final test. Set up the agent on someone else's machine. Watch them use it without help. Every question they ask reveals a gap. Every moment of confusion is a bug in your system prompt, your glossary, or your framework mappings. This isn't a demo. This is the first real run of Module 1. If it works, you've built the product LearnAIR needs. If it doesn't, your supervision log tells you exactly what to fix. Either way, you proved you can build, deploy, and supervise an AI agent system. That's the job.

Quiz

You hand the agent to a veteran. They say 'this is cool but I don't see how it connects to getting a job.' What's missing? Where in the agent do you add it?

T4

One Week Supervision Log

Use the agent daily for a week as if you were a veteran going through Module 1. Log every session: what worked, what broke, what you changed. This log IS the evidence that you can supervise an AI system. It's also the first draft of the facilitator guide: when Mike runs Module 1 with a real cohort, this log tells him what to watch for. Willison's principle: the bottleneck is validation, not building. The code is done. Now prove it works.

Quiz

After a week: how many times did you change the system prompt? How many times did you add or modify a framework mapping? What pattern do you see in what broke?
