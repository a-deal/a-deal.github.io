March 22, 2026
Two Skills Contributing to the shared hub, and standing up your own site. Both use the same tools — Git, GitHub, and Claude Code.
Part 1

Contributing to the Hub

What the hub is

A shared knowledge base at ~/src/hub/ with five directories:

DirectoryPurpose
research/Deep dives, competitive analysis, evidence
decisions/Design decisions with rationale
plans/Roadmaps, strategies, phased plans
execution/Session notes, retros, what worked
meetings/Session logs and meeting notes

How to add something

Step 1: Navigate to the hub

cd ~/src/hub

Step 2: Start Claude Code

claude

Step 3: Tell Claude what you want to create

Examples:

  • “Create a research note about [topic]. Put it in research/ dated today.”
  • “I just had a meeting with [person]. Create a meeting log in meetings/ using the template.”
  • “I found something interesting about [topic]. Add it to the hub.”

Claude will create the file with the right naming convention (YYYY-MM-DD-topic.md) and frontmatter.

Step 4: Or create it manually

touch research/2026-03-22-my-topic.md

Then open in any text editor, or tell Claude to edit it.

File template

Every file starts with frontmatter:

Template — Copy This
--- title: Your Title Here date: 2026-03-22 projects: learnair tags: research, topic1, topic2 --- # Your Title Here Content goes here. Markdown format.

What goes in the hub

What Belongs
  • Research findings from reading, listening, conversations
  • Meeting notes and takeaways
  • Ideas that cross multiple projects
  • Competitive intelligence
  • Policy or market insights
What Doesn’t Belong
  • Personal journal entries
  • Temporary task lists
  • Anything with passwords or sensitive data
  • Drafts that aren’t ready to share
  • Files belonging to other projects
Part 2

Your Own Website (GitHub Pages)

A free website at yourusername.github.io where you can publish anything: articles, a portfolio, project pages, the LearnAIR one-pager. Andrew’s is at andrewdeal.info (custom domain on top of GitHub Pages).

Prerequisites

  • GitHub account (we’ll create one)
  • Git installed (done via setup kit)
  • Claude Code (done)

Step-by-step

  1. Create a GitHub account

    Go to github.com/signup. Use your real name. Pick a username you’d put on a business card (e.g., mkdeal or mikedeal).

  2. Create the repository

    In Claude Code:

    claude -p "Help me create a GitHub Pages site. My GitHub username is [username]. Walk me through it."

    Or manually on GitHub: New Repository → name it [username].github.io (this exact format matters) → make it Public → check “Add a README file” → Create.

  3. Clone it locally
    cd ~/src git clone https://github.com/[username]/[username].github.io.git cd [username].github.io
  4. Create your first page

    Start Claude Code, then tell it what you want:

    claude

    “Create a simple index.html for my personal site. My name is Mike Deal. Keep it clean and professional. I’m a Marine veteran exploring AI workforce development.”

    Claude writes the file. You review it.

  5. Push it live
    git add . git commit -m "Initial site" git push

    Wait 2 minutes. Visit [username].github.io. Your site is live.

  6. Iterate

    Want to add a page? Tell Claude:

    • “Add a page about the LearnAIR veteran program at /learnair/”
    • “Create a blog post about what I learned this week”
    • “Add my resume as a page”

    Each time: edit locally, git add, git commit, git push. That’s the whole workflow.

Custom domain (optional, later)

  1. Buy a domain ($10–15/year) through Squarespace Domains, Namecheap, or Cloudflare
  2. In your GitHub repo settings, add the custom domain
  3. Update DNS records at your registrar
  4. Claude can walk you through the DNS setup when you’re ready

What to publish first

In order of impact:

  1. A simple landing page — who you are, what you’re about, contact info
  2. The LearnAIR one-pager — shows you’re building something real
  3. A first article — your take on AI and veterans, written from experience
  4. LinkedIn cross-post — publish on your site, share the link on LinkedIn

Common Questions

“What if I mess something up?”
Git tracks every change. You can always undo. Tell Claude “undo my last change” or run git checkout -- filename.
“Do I need to know HTML?”
No. Claude writes it. You review it and say “change this” or “make that bigger.” Markdown files (.md) are even simpler: just plain text with some formatting.
“How do I know if it worked?”
After git push, wait 2 minutes, then refresh the URL. If you see your changes, it worked.
“What if Git asks for a password?”
Set up SSH keys. Tell Claude: “Help me set up SSH keys for GitHub.” One-time setup, never type a password again.
